# 通知代理

通知代理在[AI Assistant](../ai-assistant/ai-assistant-ui.md)中添加了一个对话式AI层，您可以使用它来总结、解释通知并使用自然语言对其进行操作，从而将被动信息源转换为主动助手。 在单个交换中，您可以从“我有47个通知”变为“2个需要我在星期五前批准， 3个数据集有冲突，其余的仅供参考”，而不是扫描长列表。

## 通知代理可以执行的操作

通知代理支持以下功能：

| 功能 | 描述 |
| --- | --- |
| **通知摘要和见解** | 获取通知的简要概述，明确区分需要采取行动的通知和仅提供信息的通知，并了解通知的目的以及您收到通知的原因。 |
| **操作操作** | 直接通过对话对通知执行操作 — 将其标记为已读、暂停通知或为单个或分组通知设置提醒。 |
| **首选项和设置** | 无需导航设置，即可按解决方案和类别启用或禁用通知渠道（电子邮件、Slack、应用程序内）。 |
| **导航和深层链接** | 通过深层链接直接从通知跳转到相关的产品区域。 |
| **Collaboration和共享** | 共享通知或向同一组织中的收件人发送公告。 |

## 访问通知代理

您可以通过以下方式访问通知代理：

- **从AI助手**：打开AI助手并输入任何与通知相关的提示。
- **从提示库**：选择符合要求的建议提示。

## 示例提示

请阅读以下各节，了解可与Notifications Agent一起使用的提示示例。

### 摘要和见解

- “总结一下我最近的通知。”
- “我的哪些通知需要我的操作，哪些通知仅供参考？”

### 操作

- “将所有不可操作的通知标记为已读。”
- “1小时后提醒我有关{NOTIFICATION_OF_INTEREST}通知的信息。”

### 首选项

- “为Journey Optimizer启用Slack通知。”
- “将我的每日摘要切换到每周。”

### 协作

- “与{EMAIL_ADDRESS_OF_USER_IN_YOUR_ORG}共享此通知。”

## 最佳实践

使用通知代理时，请记住以下提示：

- **明确** — 清楚地命名应用程序、通知、通知类别和渠道，以便代理可以对预期目标执行操作。
- **链提示** — 您可以要求摘要，通过关注特定通知进行跟进，然后执行操作，所有这些操作都可以在单个对话中完成。
- **回答澄清问题** — 当代理请求确认应用程序、渠道或通知时，请确保做出响应，以便正确应用更改。
